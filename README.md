# db


エントリーヘッダ
    先頭2byte エントリーサイズ(ヘッダ含む)
    続く2byte  テーブルID

List型エントリー
    エントリーヘッダ
    続く4byte 前のエントリーのアドレス
    続く4byte 次のエントリーのアドレス
    以降、テーブルによって定められたカラムデータが並ぶ

Tree型エントリー
    エントリーヘッダ
    続く4byte 親のエントリーのアドレス
    続く4byte 左の子のエントリーのアドレス
    続く4byte 右の子のエントリーのアドレス
    続く1byte 高さ
    以降、テーブルによって定められたカラムデータが並ぶ
